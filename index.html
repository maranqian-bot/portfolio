Problem Solving

트러블슈팅 기록으로 보여주는 성장

프로젝트를 진행하며 만난 오류를 재현 조건, 원인 분석, 해결 과정, 검증 결과로 나누어 기록했습니다.

결과 화면만 확인하지 않고 로그, 요청 값, DB 상태를 함께 보며 문제가 발생한 지점을 좁혀갑니다.

같은 문제가 반복되지 않도록 원인과 확인 기준을 남기는 습관을 포트폴리오에 담았습니다.

01

문제 상황과 재현 조건을 먼저 적습니다.

02

로그, 요청 값, DB 상태를 기준으로 원인을 좁힙니다.

03

수정 후 같은 조건에서 다시 검증합니다.

인증·결제·성능 등 실제 프로젝트에서 만난 버그를 재현 조건, 원인 분석, 해결, 재검증까지 정리했습니다.
